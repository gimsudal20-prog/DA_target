# Action log detail patch - 2026-04-29

## 목적
작업 이력 로그가 단순 성공/실패 요약만 보여서, 어느 캠페인/광고그룹/행에서 실패했는지 확인하기 어려운 문제를 개선했다.

## 성능 원칙
- 로그 세부화를 위해 추가 API 호출을 만들지 않는다.
- 기존 작업 루프의 응답 JSON(`results`, `details`, `warnings`, `message`)에서만 세부 내역을 추출한다.
- 로그 파일이 과하게 커지지 않도록 세부 항목은 작업당 최대 80건, 항목당 260자로 제한한다.
- 실패/경고/건너뜀 항목을 성공 샘플보다 우선 저장한다.

## 변경 내용
- 작업 로그에 `success`, `fail`, `skipped`, `total`, `detail_items` 필드 추가
- 부분 성공 상태(`partial`) 추가
- 프론트 작업 이력에서 세부사항 접힘/펼침 UI 추가
- 실패한 POST 요청도 작업 로그가 바로 새로고침되도록 수정
- 작업 로그 엑셀 다운로드에 성공/실패/건너뜀/세부내역 컬럼 추가
- 예산 변경, 캠페인 ON/OFF 응답에 대상별 상세 결과 추가

## 주의
모든 작업에서 캠페인/광고그룹 이름이 항상 보장되지는 않는다. 기존 작업 루프에서 이름을 이미 가지고 있는 경우에는 이름을 기록하고, 그렇지 않은 경우에는 ID 기준으로 기록한다.
